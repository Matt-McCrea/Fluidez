# Fluidez — Spanish, every day

A desktop-first, fully offline daily-practice app that trains **all four parts
of language learning** — review, input, comprehension, and (above all) output —
in one structured session per day. Companion to the Español drill app: that one
builds recall of forms; Fluidez builds the ability to *use* them.

## The daily session

Open `index.html` — the app **is** the session. Five stages, in the order the
research says they belong (review protects prior learning; input before output):

1. **🔁 Repasar** — spaced-repetition review of everything due today (vocab,
   idioms, and grammar points from lessons you've studied). Misses re-queue
   today and come back tomorrow; hits move further out (1→2→4→8… days).
2. **📖 Aprender** — a full grammar lesson (formation, when to use it,
   meaning contrasts, pitfalls, examples), ending with an active-recall quick
   check whose items join your review deck.
3. **👂 Comprender** — read a short passage (gloss provided), answer
   comprehension questions; one line to translate and self-check against a model.
4. **🧩 Aplicar** — cloze sentences where the right conjugation must be chosen
   from the *meaning* (time markers, subject, mood triggers), plus sentence
   transforms. Answers are computed by the conjugation engine at runtime.
5. **✍️ Producir** — real output: build a sentence, translate one, then free
   writing against a **live constraint checklist** — the app verifies the
   mechanics ("uses the preterite", "conjugated for yo", "includes ayer") as
   you type, then shows model answers to compare against for naturalness.

Same day = same session (deterministic from the date); tomorrow rotates all
content. A streak tracks consecutive days.

## The writing checker

`js/checker.js` + the morphological index in `js/engine.js`: every conjugated
form of every verb is indexed back to its verb/tense/person, so free-typed text
can be analysed and checked against declarative constraints — including
compound tenses (he comido), accent near-misses ("almost — check the accents"),
and person/tense requirements. No AI, no network: it's all derived from the
same engine that conjugates.

Constraint types: `verbForm`, `verbFormAny`, `anyVerbInTense`, `person`,
`infinitiveUsed`, `verbType`, `containsWord`, `containsAny`, `minWords`,
`maxWords`, `question`, `negation`, `regex`.

## Profiles — one app, tuned per learner

`js/profile.js` bundles pacing/pitch knobs the session reads. Set via URL
`?p=<name>` (persists) or the switcher on the hub.

| | **Beginner** | **Standard** | **Refresher** |
|---|---|---|---|
| Review | Spanish→English **multiple choice** | type EN→ES | type EN→ES |
| New words/day | 3, **easiest categories first** | 4 | 12 |
| Vocab pool | everyday cats only (no finance/bureaucracy/…) | all | all |
| Apply (cloze) | **tap the form** from a word bank | type | type |
| Produce | guided (build + translate) | full | full + paragraphs/free-write |
| Grammar pace | a lesson **repeats 3×** before advancing | 1×; level-gated | **all unlocked** day one |

Word-bank distractors are generated by the engine (other persons/tenses of the
same verb), so wrong options are always real conjugations. Give your beginner
the link `…/?p=beginner`; the refresher link is `…/?p=refresher`.

## The home hub & the everyday extras

The app opens on a **hub** (`js/app.js` is a small router; `js/hub.js` the home
screen). The daily session is one card; the others are reached from here and
return home when done:

- **Tu progreso** (`js/progress.js`) — streak/sessions/words/due tiles, syllabus
  status, an SRS "memory strength" bar chart, a 10-week activity heatmap, and
  your worst weak spots. All from localStorage; nothing server-side.
- **Escribir** (`js/write.js`) — free-writing & journal. A guided prompt (reusing
  a paragraph task's live checklist + model) or a free "diario" with soft goals
  (length + ≥2 tenses, detected by the engine). Entries save to a personal
  journal (`localStorage['fluidez.journal']`).
- **Añadir palabras** (`js/capture.js`) — capture words met in the wild; they
  enrol into the SRS (`cap:<es>`) and come back in daily Review.
- **Tus errores** (`js/errorlog.js`) — every miss (cloze/short/review reveals) is
  logged; reviewable ones enrol into the SRS and get an on-demand drill deck
  (`js/deck.js`). Feeds daily Review too.
- **Recursos** (`js/resources.js` + `data/resources.js`) — curated input &
  reference links (edit the data file freely).

**Paragraph output** lives in both the daily Produce stage (level 2+, a
`type:'paragraph'` task) and the Escribir space — connected multi-sentence
writing with the same live constraint checklist plus a word/sentence counter.

New stores (all localStorage): `fluidez.errors`, `fluidez.captured`,
`fluidez.journal`; `fluidez.progress` gains `studied` (syllabus) and `history`
(active days for the heatmap).

## Syllabus & levels

`js/lessons.js` assembles the ordered syllabus: the 13 tense lessons are
**generated** from `data/grammar-docs.js` (shared with Español — endings tables
and recall answers come from the engine, so they can't drift), interleaved with
hand-written concept lessons from `data/grammar.js` (ser/estar, por/para, …).
Each session serves the **first unstudied lesson**; all other content (passages,
cloze, writing) is gated to your current level:

| Level | Taught |
|---|---|
| 1 | presente · ser/estar · gender & articles |
| 2 | pretérito · imperfecto · pret-vs-imp · por/para |
| 3 | futuro · condicional · perfecto |
| 4 | pres. subj. · imp. subj. · imperativo · pluscuamperfecto |
| 5 | futuro perfecto · condicional perfecto · perf. de subj. |

## Adding content (plain data files, reload to see)

- `data/grammar-docs.js` — per-tense docs (shared format with Español)
- `data/grammar.js` — hand-written concept lessons
- `data/passages.js` — readings + questions (mcq / short / translate), `level`-tagged
- `data/apply.js` — cloze (answer computed by engine) + transforms, `level`-tagged
- `data/writing.js` — build / translate / write tasks with constraints + models, `level`-tagged
- `data/verbs.js`, `data/vocab.js`, `data/idioms.js` — shared with Español

## Tests — run both before accepting any content change

```
node tools/validate-content.js   # the content gate: schemas, refs, levels,
                                 # MCQ indices, models vs own constraints,
                                 # no verb form above a passage's level
node tools/test-checker.js       # the checker's own behavioral test suite
```

The content gate is what makes generated content safe: anything that references
a verb outside the dataset, mis-levels a tense, or writes a model answer that
fails its own constraints is rejected with a specific error.
